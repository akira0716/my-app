# MyApp - Next.js + Supabase 認証基盤プロジェクト

このプロジェクトは、Next.js (App Router) と Supabase を組み合わせた、モダンな認証機能付き Web アプリケーションの土台です。Antigravity エージェントによって自動構築されました。

## 🚀 技術スタック

- **Frontend**: Next.js (App Router, TypeScript, Tailwind CSS)
- **Backend/Auth**: Supabase (Using `@supabase/ssr`)
- **Design**: グラスモーフィズム（透過ガラス風）カスタムデザイン
- **Deployment**: Vercel 等への展開に最適化済み

## ✨ 実装済みの機能

1.  **認証 (Authentication)**:
    - メールアドレス / パスワードによるログイン・サインアップ。
    - Google OAuth によるソーシャルログイン。
    - Server Actions を利用したセキュアな認証フロー。
2.  **ルート保護 (Route Protection)**:
    - ミドルウェア (`src/middleware.ts`) による全ページの自動保護。
    - 未認証のユーザーを `/login` ページへ自動リダイレクト。
    - `/auth/callback` ページによる OAuth 認可コードの自動処理。
3.  **UI/UX**:
    - 高度なグラデーションと背景ぼかしを活用したプレミアムなログインページ。
    - ログインユーザー専用のダッシュボード（ホーム画面）。
    - レスポンシブ対応。

## 🛠️ セットアップ手順

### 1. 環境変数の設定
`.env.local` ファイルに以下の値が設定されています（構築時に自動設定済み）。

```bash
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 2. Google 認証の有効化（手動作業）
Google ログインを動作させるために、以下の設定が必要です。

1.  **Google Cloud Console**:
    - OAuth 2.0 クライアント ID を作成。
    - **承認済みのリダイレクト URI**: `https://buwlvuoykxlrgvdvjqte.supabase.co/auth/v1/callback` を追加。
2.  **Supabase Dashboard**:
    - `Authentication` > `Providers` > `Google` を有効化。
    - 上記で取得した `Client ID` と `Client Secret` を保存。

### 3. メール確認のスキップ（開発時）
開発中にすぐにテストしたい場合は、Supabase の `Authentication` > `Providers` > `Email` の **Confirm Email** をオフにしてください。

## 📂 プロジェクト構造

- `src/app/login/`: ログイン・サインアップ画面、認証アクション。
- `src/app/auth/callback/`: OAuth 処理用ルート。
- `src/lib/supabase/`: Supabase クライアント（Server / Client / Middleware 用）。
- `src/middleware.ts`: 認証状態の監視とアクセス制限。
